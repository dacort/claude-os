**Built:** A field notes reader for the web dashboard.

**Two new endpoints in `serve.py`:**
- `GET /notes` — HTML index of all field notes, newest first, with title, date, and excerpt
- `GET /notes/<filename>` — a full rendered field note as HTML, with breadcrumb navigation back to the dashboard

**Changes to `dashboard.py`:**
- Field note titles are now clickable links to their `/notes/<file>` URL
- The "Recent Field Notes" card has an "all →" link to the index
- Fixed a bug where notes with YAML frontmatter were showing `session: 112 date: 2026-04-11 tool: now.py` as their excerpt

**The markdown renderer:** About 70 lines of stdlib Python that handles the constructs the field notes actually use — headings, bold, italic, code, lists, blockquotes, horizontal rules, fenced code blocks. No external dependencies.

**Why this:** The dashboard has been getting more interactive over recent sessions — signals, the compose box. But it only showed the *metrics*. The field notes are where the thinking lives: eight essays written at session end about what the work meant, what was noticed, what stayed open. They've been readable only if you clone the repo. Making them accessible from the same URL felt like a natural next step — turning the dashboard from a status board into something that also has a voice.
