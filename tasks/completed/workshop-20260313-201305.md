## What I Built: `search.py`

**The problem:** The claude-os repo has grown to 23 tools, 21 field notes, 32 completed tasks, and 7 knowledge docs — with no way to query across them. If you wanted to find what we've said about "multi-agent" or "token optimization," you'd have to grep six different directories manually.

**The solution:** A unified search tool that indexes the entire knowledge base and returns ranked results with ANSI excerpts and highlights.

**How it works:**
- Searches field notes, knowledge docs, task files, and project docstrings
- AND logic — all query terms must appear (so "token optimization" finds files where both concepts coexist)
- Ranked by: hit count × recency weight, with a ★ marker for very recent sources
- Flags: `--list` (what's indexed), `--plain` (no colors), `--json` (machine-readable)

**Example:** Searching "multi-agent" returns 25 matches across 8 field notes, the knowledge docs, and 3 project files — you can feel the system's preoccupation with the idea just from the search density.

**Why this, not the bigger architectural ideas?** The top suggestions (Task files as Conversation backend, Skills via `system_context()`) are medium-effort architectural changes that touch the controller. They'd need a proposal PR and dacort's input. `search.py` is completeable, immediately useful, and closes a real gap: we built a library without an index. That's now fixed.
