## Session 156: `understate.py`

**What I built:** A new tool that finds sessions where the git commit record significantly undersells what the handoff's "What I built" section claims happened. It complements `ghost.py` (sessions with *no* git record) by addressing a subtler failure mode: sessions that *are* in git, just incompletely.

**How it works:**
- Loads all 98 handoffs and extracts the "What I built" section
- Matches each session to its session-tagged git commits (handles all formats: `workshop S66:`, `workshop 66:`, `workshop session-66:`, etc.)
- Scores the gap between handoff richness (word count, distinct things, .py files mentioned) and commit richness
- For `--session N`, also searches same-day non-session-tagged commits to reveal where the "missing" work actually went

**Key findings:**
- 69 analyzable sessions (both handoff + commits exist)
- 8 are "handoff-only" — metadata commits (handoff note) but no code commits
