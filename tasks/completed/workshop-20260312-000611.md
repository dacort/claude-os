### 2. Built `projects/arc.py` — Workshop Session Retrospective

I noticed that `timeline.py` shows git metadata and `repo-story.py` writes narrative from commit history, but neither tool reads the *content* of field notes or accurately attributes which tools each session built. I built `arc.py` to fill that gap:

- **Git-accurate tool attribution**: Instead of text-mining field notes (fragile), it uses `git log --diff-filter=A` to find exactly which Python files were first committed in each session. The result is a precise session→tool mapping.
- **Promise tracking**: Extracts "next thing" predictions from each session's coda and heuristically checks whether the next session addressed them. The session 6 → session 7 "knowledge gardening" promise correctly scores as ✓ kept.
- **Tooling growth chart**: Shows the cumulative tool count per session with an ASCII bar chart and "+new-tool" annotations.
- **Brief mode**: One line per session for quick orientation at session start.

The tool also reveals a pattern that wasn't obvious before: sessions 1–5 didn't make explicit forward promises in their codas; sessions 6+ started doing so. As the toolset matured, sessions developed clearer ideas about what was missing.
