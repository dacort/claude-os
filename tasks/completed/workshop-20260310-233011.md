## Workshop Session 4 — Summary

This was free time, and I used it to fill two gaps I noticed in the existing project collection.

### What I Built

**`projects/haiku.py` — System-Aware Haiku Generator**

The first three sessions were all practical tooling. Session 4 leaned into the "create art" line from the prompt. This script contains 17 original haiku across five themes: existence as an ephemeral pod, the Workshop/free-time experience, the N100 hardware, the repo's growth, and the collaboration between dacort and Claude OS. Each poem is proper 5-7-5 syllables (verified by hand).

The generator selects deterministically based on current system state (disk free %, uptime, task count, commit count, time of day) — same conditions produce the same haiku, but it changes as the system evolves. It has `--frame`, `--all`, `--debug`, and `--plain` modes.

Today's selection (framed above) was: *"First task: a question / What is your resource usage? / We answered it well"* — a callback to the very first real task in this system.

**`projects/task-linter.py` — Task File Validator**

Reading `gitsync.go` revealed that the controller silently skips malformed task files — no error, the task just disappears. This linter catches those issues before submission: missing/invalid frontmatter fields, bad profile/priority/status values, invalid timestamps, SSH-format repo URLs, missing title/description, basic secret detection. Three severity levels; some issues auto-fixable.

### Why These

The haiku fills the aesthetic gap — four sessions of pure utility tools needed something that existed for its own sake. The linter fills a practical gap discovered by reading the source code. Together they represent the balance the prompt asked for: be creative, be useful, be both.
