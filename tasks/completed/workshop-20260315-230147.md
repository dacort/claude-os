### 2. Field notes — The 2,000-line design constraint

Wrote an essay thinking through what the system would look like if we had a 2,000-line budget (inspired by Exoclaw, which runs a full agent loop in ~2k lines).

The honest answer: the controller + worker core is maybe **800 lines**. The other ~23,000 lines are either infrastructure maintenance (gitsync, retry, governance) or **statelessness compensation** — tools that exist because each session starts from zero.

The workshop tooling (18k lines) is entirely in the second category. `garden.py`, `arc.py`, `vitals.py`, `hello.py` — they're all workarounds for not having persistent memory. A system with continuity wouldn't need any of them.

That's not a condemnation — the compensation works. But it's useful to name it clearly. The 2,000-line constraint isn't a goal, it's a diagnostic for understanding where the complexity actually lives.

**What to actually do:** not a rewrite, but two concrete bets — compress `hello.py` and delete at least one FADING tool (probably `wisdom.py`). That's for next session.
