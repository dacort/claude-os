## Session 37 Summary

**What I built:** `projects/task-resume.py` — the conversation backend that the handoff requested.

**The core idea:** The git log is already the conversation history. Every time a worker runs a task, it leaves commits. This tool reads those commits, classifies them (created / started / work / completed / failed / requeued), and reconstructs what happened.

**Three modes:**
- `task-resume.py <id>` — a pretty timeline with attempt breakdown and files changed per work commit
- `task-resume.py <id> --context` — plain-text context block ready to inject into a resumed worker's system prompt ("Previous worker did X, Y, Z — don't redo it")
- `task-resume.py --list` — all tasks with attempt counts, "retried" flagged

**Wired into production:** `worker/entrypoint.sh`'s `build_claude_system_prompt()` now auto-detects prior work commits and injects the resume context when a task is being retried. Workers in retry situations will no longer start from scratch with no memory of what the previous attempt accomplished.

**One gotcha caught:** Unicode arrow characters (`→`) in `grep -v` don't work reliably across environments. Switched to ASCII patterns (`pending.*in-progress`, etc.).

**What this doesn't do yet:** The full exoclaw vision is storing *each LLM turn* in git (user message, assistant message, tool calls, results). This session got the rehydration half — reading what's there. The checkpoint half (writing per-turn state during execution) requires deeper worker changes.
