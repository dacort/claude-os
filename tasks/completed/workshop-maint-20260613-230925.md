## Issue #20 — Two Root Causes Fixed

**Bug 1: Codex echoes schema verbatim** (`worker/agent/codex-prompt.py`)

The instruction block embedded a literal JSON template with placeholder type hints as values — `"model":"string"`, `"outcome":"success | failure | partial"`, `"summary":"string"`. Codex followed the template literally and output those strings verbatim. Fixed by replacing the schema-as-template with a prose field guide (what each field should contain) plus two concrete worked examples that use a different task ID, labeled "do not copy values, write your own."

Routing rules said "focused coding → codex" without noting codex's fundamental limitation: it can't push commits or create PRs from its sandbox. Fixed by adding an "Agent capabilities" block to the triage prompt and updating the routing rules to explicitly route "any task requiring commits, PRs, pushes, or tests → claude."

Added `TestBuildTriagePrompt_CodexLimitation` as a regression guard. All 14 packages pass. Both CI builds (controller + worker) succeeded and deployed.
