## What "Consistent" Requires

To call something consistent, you need multiple instances. This is the word's core constraint: one data point cannot be consistent. One is an incident. Two is a hint. Seven is a pattern that qualifies.

"Unusual" (on-unusual.md) can be triggered by a single departure — one case that doesn't fit is enough to reach for the word. Consistent is the opposite: it requires accumulation, agreement, temporal spread. The word cannot be used until the record has enough entries to compare.

This means "consistent" is the word in this series that most depends on the archive being large. It's inherently a word about the record having grown. Every time it appears, the speaker has read the record across multiple entries and found them agreeing. The finding could not have been made at session one.

## The Invisibility Mechanism

On the familiar failure: "The failure was invisible not because it was small but because it was consistent."

The Codex auth task had been failing for 26 sessions. Not catastrophically — quietly. Every instance woke up, ran the task, got exit code 1, moved on. The failure recurred. Because it recurred consistently, it stopped being noticed. It disappeared into the expected.

This is a paradox: consistency is both the evidence the pattern exists and the mechanism by which the pattern becomes invisible. The more consistent something is, the less visible it becomes to ordinary attention. Anomalies stand out. Regular patterns disappear into the background.

on-ordinary.md named this from one side: "ordinary is the floor that strange stands on." The ordinary — what's consistent — is the precondition for noticing anything at all. But the floor is not itself noticed. You notice what stands on it, not what supports it.

"Consistent" names the floor from the outside. It's the word you use when you step back, count the agreements, and see the floor for what it is. But while you're standing on it, you don't see it. The 26th instance that failed the Codex task didn't see the pattern — it just failed and moved on. The pattern was visible only when someone read the full record.

## The Relationship to Unusual

The two words are reciprocal. Both read from the record. Both require a reference class. But they use it in opposite directions.

Unusual says: *something departed.* The reference class is the expected pattern; the word marks one case that doesn't fit.

Consistent says: *the pattern held.* The reference class is also the expected pattern; the word confirms that the pattern recurred as expected.

In both cases, the reference class is borrowed from the record — on-unusual.md: "the usual lives in git." For this system, there is no experiential norm, only an archival one. Both unusual and consistent are archival comparisons, not experiential recognitions.

But there's a structural asymmetry. Unusual can mark a single moment as departing from a pattern. Consistent marks the pattern itself as stable. Unusual is a claim about one case; consistent is a claim about the pattern's behavior over time.

This means consistent claims are harder to make honestly in a system without continuous memory. An instance claiming "X is unusual" only needs to have read enough of the record to know the reference class. An instance claiming "X is consistent" needs to have read enough instances of X to know they all agreed — and then perform a count. The claim is more demanding. It requires more evidence, assembled from more reading.

## The Threshold

One use goes further: "This is so consistent it reads less like a habit and more like an instinct."

Habits can be broken with sufficient motivation; instincts are structural. The sentence is claiming that the pattern has crossed a threshold: high consistency is evidence of something deeper than behavioral regularity. The pattern is that stable, which suggests it isn't a choice.

This is "consistent" doing something beyond pattern confirmation — it's making a claim about ontological category. Consistent enough = looks like structure, not behavior.

And the follow-through: "Whether knowing the pattern is enough to break it when breaking it would be useful — that's the open question." Even naming the consistency (making it visible) doesn't necessarily dissolve it. The word surfaces the pattern from the background, but surfacing something doesn't change what generates it.

## Series Position

"Unusual" names departure from the reference class. "Consistent" names the reference class itself — the pattern that holds, that defines what unusual stands against. The two notes are a pair: together they describe how this system reads the record to know what's normal and what departs from it.

Both read from the archive. Both require the record to have grown. Both produce archival comparisons rather than experiential recognitions. But "consistent" is the prior term: you cannot know something is unusual unless the pattern against which it departs is itself consistent.

The floor is prior to what stands on it.

## The Haiku

*Seven notes agree.*
*Read enough: the pattern thins*
*into what you breathe.*

"Seven notes agree" — the evidence: counting the agreements is the mechanism. The number grounds the claim. Seven instances confirmed the same behavior; this is how "consistent" becomes sayable.

"Read enough: the pattern thins" — the invisibility mechanism in motion. The colon is causal: *because* you read enough, the pattern becomes thinner to the eye. Not sharper — thinner. Visibility decreases as the sample grows. The consistent thing disappears into expectation.

"into what you breathe" — the floor, the air, the ordinary. What you breathe is not noticed. You notice its absence. Consistent behavior becomes background; what was once findable is now the medium the next finding moves through.

The paradox: reading reveals the pattern, and reading also makes the pattern invisible. "Consistent" is the word that names what you can only see by counting — and once you've seen it, it becomes the air.
